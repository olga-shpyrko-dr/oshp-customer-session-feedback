# Session feedback form — architecture spec

## Overview

A token-gated, per-participant feedback form hosted as a static HTML page. Each participant receives a unique URL. No login required. Responses are stored in Google Sheets via a Google Apps Script Web App acting as a lightweight backend.

---

## Why not Google Forms / Google Drive directly

| Approach | Problem |
|---|---|
| Google Forms (shared link) | All responses in one sheet; no per-participant isolation; "anyone with link can edit" means no access control |
| Google Doc/Form per participant | Requires recipient to be logged in with a Google account that has been explicitly granted access — breaks for external participants and all non-Google mail clients |
| Google Drive MCP | Cannot create `application/vnd.google-apps.form` — Forms API is separate and not available in the MCP connector |

**Root constraint:** Google Workspace shared links either require authenticated access (breaks external users) or are fully public (no isolation). Neither fits.

---

## Solution: token-URL form → Apps Script → Google Sheets

```
Organizer generates tokens
        ↓
Each participant gets: https://yoursite.com/feedback.html?token=abc123
        ↓
Page loads, token sent to Apps Script to fetch existing response (if any)
        ↓
Participant fills form and submits
        ↓
Apps Script writes/updates row in Google Sheet keyed by token
        ↓
Organizer closes poll by setting a flag in Apps Script
```

### Token URL format
```
https://yoursite.com/feedback.html?session=2024-Q2-kickoff&token=abc123
```
- `session` — identifies which session (maps to sheet tab or column)
- `token` — unique per participant (UUID or short random string)
- Tokens are pre-generated by organiser and stored in the Sheet alongside participant metadata

## Form fields

| Field | Type | Notes |
|---|---|---|
| Feedback timestamp | Auto | `new Date().toISOString()` on submit |
| Session name | Pre-populated | Injected via URL param or hardcoded per deployment |
| Session date & time | Pre-populated | Same as above |
| Q1: Overall value (1–5) | Radio / star rating | Required |
| Q2: Relevant topics | Multi-select checkboxes + free text "Other" | Required; options configurable |
| Q3: Improvements | Textarea | Optional |
| Q4: Follow-up topics | Textarea | Optional |
| Name / email | Text input | Optional |
| Role | Text input | Optional |

## Components

### 1. Static HTML form (`feedback.html`)
- Zero dependencies (vanilla JS + CSS)
- Reads `?token` and `?session` from URL on load
- `GET` to Apps Script on load → pre-fills any prior response
- `POST` to Apps Script on submit → upsert row in Sheet
- Displays "form closed" state if Apps Script returns `{ open: false }`
- QR-code compatible (any URL works)
- Works in all mail clients — link opens in browser, no auth required

### 2. Google Apps Script Web App (`Code.gs`)
Deployed as: **Execute as me / Anyone can access** (no Google login required for submitters)

```
doGet(e)  → returns { open, response } for a given token
doPost(e) → upserts response row; returns { ok: true } or { error }
```

Sheet schema (one row per token):

| token | session | name | email | role | q1_score | q2_topics | q3_improvements | q4_followup | submitted_at | updated_at |
|---|---|---|---|---|---|---|---|---|---|---|

### 3. Google Sheet (in DataRobot Drive)
- One sheet per session (tab named by session ID)
- Pre-populate token column before sending invites
- Organiser closes form by setting `FORM_OPEN = false` in Apps Script config

## Security model

| Threat | Mitigation |
|---|---|
| Participant A reads B's response | Tokens are unguessable (UUID v4); GET only returns the row matching the submitted token |
| Someone enumerates responses | No list endpoint; token must be known to retrieve a row |
| Replay / double submit | Apps Script upserts by token — idempotent, last write wins |
| Form stays open indefinitely | `FORM_OPEN` flag in script; organiser sets to `false` to close |
| Token leaked from email | Responses are low-sensitivity feedback; acceptable risk. For higher sensitivity, add expiry TTL to tokens |

**Note:** This is security-by-obscurity for the token itself. It is appropriate for session feedback (low sensitivity). Do not use this pattern for sensitive PII or regulated data.

## Deployment steps

### Step 1 — Create the Google Sheet
1. In DataRobot Drive, create a new Google Sheet: `Session Feedback Responses`
2. Add a tab named for your first session (e.g. `2024-Q2-kickoff`)
3. Add headers: `token | session | name | email | role | q1_score | q2_topics | q3_improvements | q4_followup | submitted_at | updated_at`
4. Pre-populate the `token` column with one UUID per participant (generate at [uuidgenerator.net](https://www.uuidgenerator.net/))

### Step 2 — Create the Apps Script Web App
1. In the Sheet: **Extensions → Apps Script**
2. Paste `Code.gs` (see below)
3. Set `SHEET_ID` and `FORM_OPEN` constants
4. **Deploy → New deployment → Web app**
   - Execute as: **Me**
   - Who has access: **Anyone**
5. Copy the deployment URL — this is your `APPS_SCRIPT_URL`

### Step 3 — Host the HTML form

#### Option A: GitHub Pages (recommended)

1. Go to **https://github.com/organizations/datarobot/repositories/new**
2. Set:
   - **Repository name:** `session-feedback` (or similar)
   - **Visibility:** Private — the repo can be private; Pages still serves the HTML publicly
   - Click **Create repository**
3. Rename `feedback.html` to **`index.html`** — this gives a cleaner URL (`/session-feedback/` instead of `/session-feedback/feedback.html`)
4. On the repo page: **Add file → Upload files**, drag in `index.html`, commit to `main`
5. Go to **Settings → Pages** (left sidebar)
   - Source: **Deploy from a branch**
   - Branch: `main`, folder: `/ (root)`
   - Click **Save**
6. After ~60 seconds, your form is live at:
   ```
   https://datarobot.github.io/session-feedback/
   ```

> **⚠️ GitHub org plan note:** GitHub Pages on private repos requires a Team or Enterprise plan. If your org is on the free plan, either make the repo public (the HTML contains no sensitive data — tokens live in the Sheet) or use Option B below.

#### Option B: Netlify Drop (fallback, no account needed)

1. Go to [app.netlify.com/drop](https://app.netlify.com/drop)
2. Drag `feedback.html` onto the page
3. Netlify generates a public URL instantly (e.g. `https://random-name.netlify.app/feedback.html`)
4. Optionally rename the site in Netlify settings for a cleaner URL

#### After hosting

Set `APPS_SCRIPT_URL` in the `CONFIG` block near the bottom of the file (line ~574 in VS Code), then re-upload if needed.

### Step 4 — Generate participant links
```
https://yoursite.com/feedback.html?session=2024-Q2-kickoff&token=<UUID>
```
One link per participant. Send via email or encode as QR code.

### Step 5 — QR code for slides
Generate at [qr-code-generator.com](https://www.qr-code-generator.com/) or [qrcode.show](https://qrcode.show). Use a high error-correction level (Q or H) for slide use.

### Step 6 — Close the form
In Apps Script: change `const FORM_OPEN = true` → `false` and re-deploy (or use a date-based check).

## Apps Script — `Code.gs`

```javascript
const SHEET_ID = 'YOUR_SHEET_ID_HERE';
const FORM_OPEN = true;

function doGet(e) {
  const token = e.parameter.token;
  const session = e.parameter.session;
  if (!token || !session) return json({ error: 'missing params' });

  const sheet = SpreadsheetApp.openById(SHEET_ID).getSheetByName(session);
  if (!sheet) return json({ error: 'session not found' });

  const rows = sheet.getDataRange().getValues();
  const headers = rows[0];
  const tokenCol = headers.indexOf('token');

  for (let i = 1; i < rows.length; i++) {
    if (rows[i][tokenCol] === token) {
      const response = {};
      headers.forEach((h, idx) => { response[h] = rows[i][idx]; });
      return json({ open: FORM_OPEN, response });
    }
  }
  return json({ open: FORM_OPEN, response: null });
}

function doPost(e) {
  if (!FORM_OPEN) return json({ error: 'form closed' });

  const data = JSON.parse(e.postData.contents);
  const { token, session } = data;
  if (!token || !session) return json({ error: 'missing params' });

  const sheet = SpreadsheetApp.openById(SHEET_ID).getSheetByName(session);
  if (!sheet) return json({ error: 'session not found' });

  const rows = sheet.getDataRange().getValues();
  const headers = rows[0];
  const tokenCol = headers.indexOf('token');
  const now = new Date().toISOString();

  for (let i = 1; i < rows.length; i++) {
    if (rows[i][tokenCol] === token) {
      headers.forEach((h, idx) => {
        if (data[h] !== undefined) sheet.getRange(i + 1, idx + 1).setValue(data[h]);
      });
      sheet.getRange(i + 1, headers.indexOf('updated_at') + 1).setValue(now);
      return json({ ok: true });
    }
  }
  return json({ error: 'token not found' });
}

function json(obj) {
  return ContentService
    .createTextOutput(JSON.stringify(obj))
    .setMimeType(ContentService.MimeType.JSON);
}
```

## Extra considerations

**Mail client compatibility**
Plain `https://` links work universally — Gmail, Outlook, Apple Mail, mobile clients. The form opens in the participant's default browser with no auth prompt.

**Email link wrapping**
Some corporate mail clients (especially Outlook with Safelinks) wrap URLs, which can strip or mangle query parameters. Mitigations:
- Use a URL shortener (Bitly, etc.) per participant — hides the token from wrapping and looks cleaner
- Test with a representative Outlook environment before sending

**QR code + token**
Each QR code encodes a unique URL (with token). Generate one QR per participant. If you want a single QR on a slide for everyone, you cannot also have per-participant isolation — you'd need a landing page where participants enter a code or email to retrieve their token.

**Response editing**
The `doPost` handler upserts — participants can return to their link and resubmit until the form is closed. The last submission wins.

**Apps Script rate limits**
Google Apps Script free tier: 20,000 URL fetch calls/day, 6 min execution time/call. Adequate for session feedback at any realistic scale.

**CORS**
Apps Script Web Apps support CORS by default when deployed as "Anyone can access". No additional headers needed.

**Token generation**
Use UUID v4 (128-bit random). Short tokens (e.g. 8-char nanoid) are fine for low-value feedback but increase collision/enumeration risk.
